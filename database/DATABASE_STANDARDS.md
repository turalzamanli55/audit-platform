# Database Standards

## Purpose

Enterprise database conventions for the Audit Platform foundation layer. Authoritative for Sprint 2B and all subsequent schema work.

## Status

Active — Sprint 2B

## Naming Convention

| Object | Convention | Example |
|--------|------------|---------|
| Tables | `snake_case`, plural nouns | `organizations`, `role_permissions` |
| Columns | `snake_case` | `organization_id`, `created_at` |
| Primary keys | `id` (UUID) | `id uuid PRIMARY KEY` |
| Foreign keys | `{referenced_table_singular}_id` | `organization_id` |
| Indexes | `{table}_{columns}_{purpose}_idx` | `workspaces_organization_id_idx` |
| Unique indexes | `{table}_{columns}_active_uidx` | `organizations_slug_active_uidx` |
| Triggers | `trg_{table}_{action}` | `trg_organizations_set_updated_at` |
| Functions | `snake_case`, verb-led | `set_updated_at`, `user_belongs_to_organization` |
| Enums | `snake_case` type, lowercase values | `record_status` → `active` |

## UUID Strategy

- All primary keys use `uuid` generated by `gen_random_uuid()` (requires `pgcrypto`).
- Foreign keys reference UUID primary keys explicitly.
- No sequential integer IDs for tenant data.
- Auth user references use `auth.users(id)`.

## Timestamp Strategy

- All timestamps are `timestamptz` stored in UTC.
- `created_at` defaults to `public.utc_now()` and is immutable after insert.
- `updated_at` is maintained by `trg_*_set_updated_at` trigger calling `set_updated_at()`.
- Soft delete uses `deleted_at timestamptz` (NULL = active).

## Soft Delete Strategy

- Physical deletes are prohibited for tenant entities in application code.
- `deleted_at IS NULL` defines active records.
- Partial unique indexes include `WHERE deleted_at IS NULL`.
- Repositories filter with `.is("deleted_at", null)`.
- `deleted_by` records the acting user UUID.

## Audit Columns

Every foundation table includes:

| Column | Purpose |
|--------|---------|
| `created_by` | Actor at insert (default `auth.uid()` via trigger) |
| `updated_by` | Actor at last update |
| `deleted_by` | Actor at soft delete |

Triggers `set_created_by` and `set_updated_by` populate audit actors automatically.

## Versioning Strategy

- `version integer NOT NULL DEFAULT 1` on every table.
- `set_row_version()` increments version on each UPDATE.
- Application updates must include expected version for optimistic locking.

## Optimistic Locking Strategy

```sql
UPDATE organizations
SET name = $1
WHERE id = $2 AND version = $3 AND deleted_at IS NULL;
```

Repositories pass `expectedVersion` and use `.eq("version", expectedVersion)`. A zero-row update surfaces as `NotFoundError` or `ConflictError`.

## Index Strategy

1. **Primary keys** — clustered UUID PK on every table.
2. **Foreign keys** — B-tree index on every FK column used in joins or RLS.
3. **Unique constraints** — partial unique indexes scoped to active rows (`deleted_at IS NULL`).
4. **Search** — GIN `to_tsvector` indexes on `name` columns for organizations, workspaces, companies.
5. **Audit** — `audit_logs(created_at DESC)` for chronological queries.

## Foreign Key Strategy

| Relationship | ON DELETE |
|--------------|-----------|
| Tenant parent → child | `RESTRICT` (prevent orphan cascade) |
| `memberships.user_id` → `auth.users` | `CASCADE` |
| Audit actor columns → `auth.users` | `SET NULL` |

No circular FK dependencies. Hierarchy: `organizations` → `workspaces` → `companies`.

## Constraint Strategy

- `NOT NULL` on required business fields.
- `CHECK` constraints for non-empty strings (`char_length(trim(...)) > 0`).
- Scope consistency checks (e.g. platform roles require `organization_id IS NULL`).
- Enum types for `status`, `scope`, and membership scope.

## Migration Strategy

- Sequential timestamped files in `supabase/migrations/`.
- One concern per migration where practical:
  1. Extensions and common functions
  2. Foundation tables
  3. RLS policies
- Seed data in `supabase/seed.sql` (roles, permissions only — no tenant data).
- Apply via `supabase db push` or `supabase db reset --local`.

## Rollback Strategy

- Forward-only migrations in production.
- Rollback = new compensating migration (never edit applied migrations).
- Document breaking changes in migration header comments.
- Test locally with `supabase db reset` before push.

## Enterprise Column Standard

Every foundation table includes:

```
id, created_at, updated_at, created_by, updated_by, deleted_at, deleted_by, version, status
```

Plus domain-specific columns.
